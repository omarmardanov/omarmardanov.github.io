# Чек-лист перед сдачей

Проходить на каждом материале перед тем, как пометить его «готов».

---

## Общее (любой тип материала)

- [ ] В шапке указаны: тип, slug, автор, дата, статус.
- [ ] `<title>` ≤ 65 символов (для кейса — ≤ 70).
- [ ] Meta description 130–160 символов, не начинается с «На этой странице…».
- [ ] H1 содержит italic-акцент через `*звёздочки*`.
- [ ] Lead под H1 отвечает на «кому + что делаем» (для услуги/направления) или «какая проблема + что узнает читатель» (для статьи).
- [ ] Прошёлся по чёрному списку штампов — все убрал (см. [01-Правила и стиль](01-Правила%20и%20стиль.md), §2).
- [ ] Все цифры конкретные. Никаких «много», «значительно», «более десяти».
- [ ] Все сроки в днях/неделях/месяцах. Не «быстро», не «оперативно».
- [ ] Кавычки «ёлочки», тире длинное `—` между предложениями, среднее `–` в диапазонах.
- [ ] Внутренние ссылки — на существующие URL.
- [ ] Фото прислано или помечено «подобрать».

---

## Дополнительно — услуга

- [ ] Facts strip: 3 ячейки заполнены конкретными цифрами.
- [ ] Aside-nav: 4–6 пунктов, совпадают с H2.
- [ ] H2 «Когда нужна» — заполнено.
- [ ] H2 «Что входит» — 5–10 конкретных пунктов, не «консультирование».
- [ ] H2 «Стоимость» — таблица с разделением «услуги бюро» / «госпошлины».
- [ ] H2 «FAQ» — 5–8 реальных вопросов клиентов.
- [ ] Цены: `от X ₽` / `X – Y ₽` / `по запросу`. Не «доступные», не «договорная».

---

## Дополнительно — направление

- [ ] 3–6 разделов, в каждом 4–10 услуг.
- [ ] У каждой услуги есть slug.
- [ ] Aside-nav: 3–6 пунктов, совпадают с разделами.
- [ ] CTA-final сформулирован под контекст направления.

---

## Дополнительно — статья

- [ ] Длина тела 5 000–15 000 знаков с пробелами.
- [ ] 3–5 H2-разделов.
- [ ] Минимум одно из: H3 / список / blockquote / callout «Что важно знать».
- [ ] 2–5 внутренних ссылок на услуги/кейсы/статьи/МКТУ.
- [ ] Все цитаты — с источником.
- [ ] Под статьёй: автор (имя + роль + фото 200×200).
- [ ] Cover 1600×700 JPG прислан или помечено «подобрать».
- [ ] Указана рубрика — одна из существующих, новую не выдумываем.

---

## Дополнительно — кейс

- [ ] Согласие клиента есть и сохранено (PDF), либо кейс анонимизирован.
- [ ] Категория (1–3 через `·`) указана.
- [ ] Номер дела указан, если публичный.
- [ ] Метка результата (1–3 слова) — однозначная.
- [ ] Значение результата — конкретная цифра.
- [ ] H2 «Задача» 200–500 слов.
- [ ] H2 «Решение» 300–700 слов + список 4–8 ключевых действий.
- [ ] H2 «Результат» 100–300 слов + 3 метрики.
- [ ] Cover 1600×900 JPG прислан или помечено «подобрать».

---

## Дополнительно — сотрудник

- [ ] ФИО в карточке: Имя + Фамилия (без отчества).
- [ ] Должность — одна строка, без `·` и перечислений.
- [ ] Статус (если есть) — по реестровому стандарту.
- [ ] Опыт указан только если 3+ лет.
- [ ] Город — Москва или Тюмень.
- [ ] Направление — максимум 2 темы через запятую.
- [ ] Фото 1000×1000 JPG на сером фоне `#F0EEEA`, имя файла = `{Фамилия латиницей}.jpg`. Либо помечено «инициалы на время».
- [ ] Не добавил биографию, образование, награды, хобби — это всё не помещается на карточку.
