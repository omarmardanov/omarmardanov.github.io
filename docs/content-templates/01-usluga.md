# Шаблон: страница услуги

> Шаблон для **одной услуги** — например, «Регистрация товарного знака», «Возражение
> против отказа Роспатента», «Договор франшизы». Всего таких страниц будет 110+.
>
> **Эталон на сайте:** [/uslugi-shablon/](https://1-tm.ru/uslugi-shablon/)
> **URL результата:** `/uslugi/{направление}/{раздел}/{slug}/`
>
> Прочитайте сначала [00-pravila-i-stil.md](00-pravila-i-stil.md) — про тон, цифры,
> курсив, чёрный список штампов. Здесь — только структура полей.

---

## 0. Шапка документа

```
Тип: услуга
Slug (URL): [например, registratsiya-tovarnogo-znaka]
Направление: [интеллектуальная собственность / договоры и сделки / споры и защита / сопровождение бизнеса]
Раздел: [например, товарные знаки]
Автор брифа: [Фамилия И.]
Дата: [ГГГГ-ММ-ДД]
Статус: черновик
```

## 1. SEO-поля

### 1.1 `<title>` (до 65 символов)

```
[Название услуги] · Галифанов, Мальков и Партнёры
```

**Пример:** `Регистрация товарного знака · Галифанов, Мальков и Партнёры`

### 1.2 Meta description (130–160 символов)

```
[Что за услуга в 1 фразе] + [ключевая выгода или для кого] + [конкретика: срок или цифра].
```

**Пример:**
`Полный цикл регистрации товарного знака в Роспатенте: предварительный поиск, заявка, делопроизводство. Сроки 6–10 мес., успешность 96%.`

## 2. Hero страницы

### 2.1 Eyebrow (моно-метка над H1, 3–6 слов)

```
[Раздел · Направление]
```

**Пример:** `Товарные знаки · Интеллектуальная собственность`

### 2.2 H1 (40–80 символов, с italic-акцентом)

```
[Название услуги] *[акцент]*
```

**Примеры:**
- `Регистрация товарного знака *в Роспатенте*`
- `Возражение против отказа *экспертизы Роспатента*`
- `Патентный поиск *перед подачей заявки*`

Курсив — на ключевом смысловом отличии. См. правила §4.

### 2.3 Lead-абзац под H1 (150–400 символов, 1–3 предложения)

```
[Что делаем + кому подходит + ключевая выгода или конкретика].
```

**Пример:**
`Полный цикл регистрации в Роспатенте — от предварительного поиска до получения свидетельства. Подходит малому и среднему бизнесу, выводящему новый бренд на рынок. Срок 6–10 месяцев, успешность 96%.`

### 2.4 Кнопки в hero

- **Главная (всегда):** `Заявка на консультацию` — открывает модалку (универсально, не менять).
- **Вторая (опционально):** ссылка-вопрос на якорь страницы.
  Примеры: `Как мы работаем →` (→ `#etapy`), `Узнать стоимость →` (→ `#stoimost`).

---

## 3. Facts strip (3 ячейки, обязательно)

Три ячейки с конкретикой. Каждая = метка (≤ 15 символов) + значение (≤ 25 символов).

| # | Метка | Значение | Пример |
|---|---|---|---|
| 1 | Стоимость | от X ₽ | `от 35 000 ₽` |
| 2 | Срок | X мес. / недель | `6–10 мес.` |
| 3 | Опыт бюро | N+ единиц | `1 800+ знаков` |

Альтернативные варианты ячейки 3 (если опыт «знаками» нерелевантен):
- `Успешность: 96%`
- `Без отказа: 9 из 10`
- `С 2014 года`

---

## 4. Aside-nav (содержание справа, 4–6 пунктов)

Названия H2-секций, которые есть на этой странице. Должны быть короткими (1–3 слова), это **меню навигации**, не заголовки.

**Пример для регистрации ТЗ:**
1. Когда нужна
2. Что входит
3. Этапы
4. Стоимость
5. Частые вопросы

---

## 5. Aside-card (контактный блок справа)

Один короткий призыв с фокусом на эту конкретную услугу — **не** общий «Закажите консультацию».

```
Заголовок: [Короткий offer, 5–10 слов]
Текст: [1–2 предложения о том, что произойдёт после клика]
Кнопка: [Действие в 2–3 словах]
```

**Пример:**
> **Бесплатная оценка шансов**
> Пришлите название или эскиз — за 1 рабочий день оценим вероятность регистрации.
> [Прислать обозначение]

## 6. Контентные секции (H2)

Обязательны разделы 6.1, 6.2, 6.5. Разделы 6.3 и 6.4 — настойчиво рекомендуются. Можно добавлять свои H2 при необходимости — лимит 4–6 H2-секций на страницу.

### 6.1 H2: «Когда нужна услуга» (обязательно)

**Длина:** 200–500 слов. 1–3 абзаца + опционально маркированный список из 3–6 пунктов «Типичные ситуации».

**Что писать:**
- Триггеры: что должно произойти у клиента, чтобы он пришёл за этой услугой.
- Признаки, что пора обращаться (а не «когда-нибудь потом»).
- Риски бездействия — конкретно, без запугивания.

**Заполните:**
```
[Абзац 1 — общая ситуация, 50–100 слов]

[Абзац 2 — для кого особенно важно, 50–100 слов]

Типичные триггеры обратиться:
- [триггер 1]
- [триггер 2]
- [триггер 3]
- [триггер 4]
```

### 6.2 H2: «Что входит» (обязательно)

**Длина:** 5–10 пунктов списка. Опционально краткий вводный абзац (50–100 слов).

**Формат:** маркированный или нумерованный список. Каждый пункт — **конкретная работа**, не «консультирование» и не «помощь».

**Хороший пункт:**
✅ `Поиск по базе Роспатента (54 000+ обозначений) с подготовкой отчёта по тождественным и сходным знакам`
❌ `Профессиональная консультация по вопросам регистрации`

**Заполните:**
```
[Вводный абзац, опционально, 50–100 слов]

- [работа 1]
- [работа 2]
- [работа 3]
- [работа 4]
- [работа 5]
- ...
```

### 6.3 H2: «Этапы работы» (рекомендуется)

**Длина:** 3–7 этапов. Каждый этап — название (3–6 слов) + описание (1–2 предложения, 80–150 символов) + срок.

**Формат:** нумерованный список.

**Заполните:**
```
01. **[Название этапа]** — [что именно делаем + что от клиента]. Срок: [N дней/недель].
02. **[Название этапа]** — [...]. Срок: [N дней/недель].
03. **[Название этапа]** — [...]. Срок: [N дней/недель].
```

**Пример:**
> 01. **Анализ обозначения** — проводим поиск по базе Роспатента, оцениваем шансы регистрации, формулируем перечень рисков. От вас — обозначение в любом формате. Срок: 3–5 рабочих дней.
> 02. **Подготовка заявки** — формируем перечень товаров и услуг по МКТУ, готовим описание, оплачиваем пошлину. Срок: 5–7 рабочих дней.
> 03. **Подача и ведение делопроизводства** — направляем заявку в Роспатент, отвечаем на запросы экспертизы, при необходимости подаём возражения. Срок: 6–9 месяцев.

### 6.4 H2: «Стоимость и пошлины» (рекомендуется)

**Длина:** таблица из 3–8 строк + итоговая строка.

**Формат:** разбивка по статьям. **Услуги бюро отдельно, госпошлины отдельно** — клиент должен видеть структуру.

**Заполните:**
```
| Статья | Сумма |
|---|---|
| Услуги бюро | от X ₽ |
| Пошлина за регистрацию заявки | X ₽ |
| Пошлина за экспертизу (1 класс МКТУ) | X ₽ |
| [Дополнительные пошлины при необходимости] | X ₽ |
| **Итого** | **от X ₽** |
```

После таблицы — 1 абзац (50–150 слов) с пояснением: что меняет цену (дополнительные классы МКТУ, ускоренное рассмотрение, экспертные исследования, и т.д.).

### 6.5 H2: «Частые вопросы» (обязательно)

**Длина:** 5–8 вопросов. Каждый ответ — 1–4 предложения (50–250 символов).

**Что писать:** реальные вопросы клиентов. **Не маркетинговые** («Зачем регистрировать товарный знак?» — это вопрос статьи, не FAQ услуги).

**Заполните:**
```
**[Вопрос 1]?**
[Ответ, 1–4 предложения.]

**[Вопрос 2]?**
[Ответ.]
```

**Примеры хороших FAQ-вопросов:**
- Можно ли подать заявку самостоятельно?
- Что делать, если уже получили отказ?
- Защищает ли регистрация в РФ за рубежом?
- Можно ли зарегистрировать домен как товарный знак?
- Сколько классов МКТУ выбрать?
- Можно ли передать товарный знак другой компании?

## 7. Связанные блоки (заполнит редактор, не вы)

Внизу страницы автоматически идут блоки «Связанные услуги» и «Связанные дела». Редактор сайта подбирает их из других материалов того же направления. Вы можете предложить, но не обязаны.

**Если хотите предложить:**
- Связанные услуги (3–4 штуки): slugs других страниц-услуг.
- Связанные дела (3 штуки): slugs кейсов по теме.

## 9. Чек-лист

- [ ] Шапка документа заполнена (тип, slug, направление, раздел, автор, дата, статус)
- [ ] `<title>` ≤ 65 символов с пробелами
- [ ] Meta description 130–160 символов, начинается не с «На этой странице…»
- [ ] H1 содержит italic-акцент через `*звёздочки*`
- [ ] Lead 150–400 символов, отвечает на «что делаем + для кого»
- [ ] Facts strip — 3 ячейки заполнены конкретными цифрами
- [ ] Aside-nav — 4–6 коротких пунктов, совпадают с H2
- [ ] Aside-card — короткий призыв конкретно по этой услуге
- [ ] H2 «Когда нужна» — заполнено
- [ ] H2 «Что входит» — 5–10 конкретных пунктов
- [ ] H2 «Этапы» — заполнено (если применимо)
- [ ] H2 «Стоимость» — таблица с разделением бюро/пошлины (если применимо)
- [ ] H2 «FAQ» — 5–8 вопросов с короткими ответами
- [ ] Прошёл по чёрному списку штампов (см. правила §2)
- [ ] Все цены в формате `от X ₽` / `X – Y ₽` / `по запросу`
- [ ] Все сроки конкретные (не «быстро»)
- [ ] Внутренние ссылки на другие услуги/кейсы — корректные slugs
