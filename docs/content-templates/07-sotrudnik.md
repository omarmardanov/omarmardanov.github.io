## Структура карточки

Карточка содержит ровно 4–5 элементов и **больше ничего**:

1. Фото-портрет (1000×1000, серый фон)
2. ФИО
3. Должность
4. До 3 строк дополнительной информации (статус, опыт+город, направление)

Не пытайтесь добавить туда биографию, описание характера, увлечения и прочее —
карточка маленькая, всё лишнее ломает дизайн. Если нужно развернутее — это
повод для **отдельной статьи** в журнале о сотруднике, не карточки.

## 1. Поля

### 1.1 ФИО (только имя и фамилия в карточке)

```
[Имя Фамилия]
```

**Правило:** в карточке — Имя + Фамилия. Отчество не пишем (не помещается визуально). Если есть привычка коммуникации с отчеством — указать отдельно в досье для WP, но в карточку идёт без него.

**Примеры:**
- ✅ `Дмитрий Галифанов`
- ❌ `Галифанов Дмитрий Юрьевич`
- ❌ `Д. Ю. Галифанов`

### 1.2 Должность (только должность)

```
[Должность]
```

**Правило:** одна строка, **только должность**. Без `·`, без перечислений, без региона, без статуса. Всё остальное идёт в `creds` (поле 1.3).

**Примеры:**
- ✅ `Партнёр`
- ✅ `Старший юрист`
- ✅ `Юрист`
- ✅ `Финансовый директор`
- ❌ `Партнёр · Патентный поверенный РФ` ← статус идёт в creds
- ❌ `Старший юрист направления "Товарные знаки"` ← направление идёт в creds
- ❌ `Юрист, г. Тюмень` ← город идёт в creds

### 1.3 Доп. строки (creds, до 3 строк в строгом порядке)

3 строки в **этом порядке**:

#### Строка 1: Статус (опционально)

Профессиональный статус, удостоверение, реестровое членство.

**Варианты:**
- `Патентный поверенный РФ`
- `Адвокат · АП Москвы`
- `Бывший сотрудник ФАС России`
- `Налоговый консультант (Реестр Палаты налоговых консультантов)`
- `Член Палаты по патентным спорам`

Если статуса нет — **строку пропускаем**, не оставляем пустую.

#### Строка 2: Опыт + город (одной строкой)

Формат: `Опыт — N лет · Город`

**Примеры:**
- `Опыт — 12 лет · Москва`
- `Опыт — 8 лет · Тюмень`
- `Опыт — 25 лет · Москва`

**Правила:**
- Опыт указываем только если он 3+ лет. Меньше 3 — не указываем (пропускаем строку).
- Город — основной офис сотрудника (Москва или Тюмень).

#### Строка 3: Направление (1 направление, опционально)

Профильное направление работы. Максимум 2 темы через запятую.

**Примеры:**
- `Товарные знаки, патенты`
- `Арбитражные споры, СИП`
- `Налоги, договоры`
- `Сопровождение бизнеса`
- `Управление, финансы` (для не-юристов вроде фин. директора)

### Заполнение в шапке документа

```
Имя: [Дмитрий]
Фамилия: [Галифанов]
Отчество (для WP, не на карточку): [Юрьевич]
Должность: [Партнёр]
Статус (опц.): [Патентный поверенный РФ]
Опыт — лет (опц., если 3+): [25]
Город: [Москва]
Направление (опц.): [Товарные знаки, патенты]
```

## 2. Фото

### 2.1 Технические требования

- **Размер:** 1000×1000 пикселей, аспект ровно 1:1
- **Формат:** JPG, sRGB, качество 80–85%, без EXIF
- **Фон:** серый `#F0EEEA` (paper-2) — **критично**, иначе карточка выбьется из дизайна
- **Имя файла:** фамилия латиницей в Title Case, например: `Galifanov.jpg`, `Malkov.jpg`, `Kolesnikov.jpg`

### 2.2 Композиция фото

- В кадре: голова и плечи (head-and-shoulders).
- Фронтальный поворот или лёгкий ¾.
- Освещение: фронтальное, мягкое. Без жёстких теней.
- Деловой стиль одежды.
- Нейтральное выражение или лёгкая улыбка (не зубастая, не «корпоративная»).

### 2.3 Что делать, если фото пока нет

Не присылать заглушку из стоков и не использовать селфи. Лучше прислать
данные сотрудника **без фото** — на карточке отобразятся инициалы
(2 буквы кириллицы на сером фоне), что вполне приемлемо как временный
вариант.

Когда появится реальное фото — отправить отдельно, разработчик заменит.

### 2.4 Где хранятся файлы фото

```
assets/images/team/
  Galifanov.jpg
  Malkov.jpg
  Kolesnikov.jpg
  ...
```

Присылать в `content/photos/team/`.

## 4. Что НЕ писать в карточку

Эти данные **не помещаются** и **не нужны** на карточке. Если хотите, чтобы это было на сайте — пишите отдельную страницу/статью.

- Биография, истории «начал карьеру в…»
- Перечисление образования и вузов
- Награды и медали (исключение — статус из реестра, см. строку 1)
- Языки
- Хобби, увлечения
- Цитаты сотрудника
- Email/телефон сотрудника лично (общий контакт бюро — в футере)
- Ссылки на соцсети сотрудника
- Длинный список услуг, которые делает

---

## 5. Что прислать вместе с данными

1. Шапка документа (поля выше) — Word/Google Doc на сотрудника или одна общая таблица на всю команду.
2. JPG-портрет в `content/photos/team/{Фамилия}.jpg` — 1000×1000, серый фон.

Если данные по нескольким сотрудникам — лучше **одной таблицей** (Excel/Sheets), фото — отдельной папкой.

## 6. Чек-лист (на каждого сотрудника)

- [ ] ФИО в карточке: только Имя + Фамилия (без отчества)
- [ ] Должность: одна строка, **только должность** (без `·`, без статуса)
- [ ] Статус (если есть) — формулировка по реестровому стандарту
- [ ] Опыт указан только если 3+ лет
- [ ] Город — Москва или Тюмень
- [ ] Направление — максимум 2 темы через запятую
- [ ] Фото 1000×1000 JPG на сером фоне `#F0EEEA` ИЛИ помечено «инициалы на время»
- [ ] Имя файла фото = `{Фамилия латиницей}.jpg`
- [ ] Пройдено: не пишем хобби, биографию, награды
