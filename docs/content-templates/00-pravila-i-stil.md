# Правила и стиль — единый гайд по контенту 1-tm.ru

> Этот документ — основа для всех текстов на сайте. Шаблоны (`01–07-*.md`) опираются
> на правила отсюда. Перед заполнением любого шаблона прочитайте разделы 1–4.
> Разделы 5–9 — справочник, к которому возвращаетесь по мере работы.

**Кто читает этот гайд:** юристы, маркетологи, ассистенты партнёров, копирайтеры — все, кто пишет тексты для сайта.
**Кто получает готовый текст:** WP-разработчик (загружает в админку без переписывания).

---

## 1. Тон голоса (3 строки, важнее всего)

1. **Сухо, конкретно, по делу.** Юрист, разговаривающий с клиентом-собственником бизнеса. Не маркетолог, не лектор.
2. **Цифра побеждает прилагательное.** «12,4 млн ₽ компенсации» вместо «крупная компенсация». «8 месяцев» вместо «быстро».
3. **«Мы» / «бюро», обращение на «вы».** Не «компания», не «фирма», не «наша команда».

Если предложение можно сказать в баре пиво пьющему директору без того, чтобы он закатил глаза — оно пройдёт. Если звучит как пресс-релиз — переписываем.

## 2. Чёрный список штампов

Эти слова и фразы **не пишем никогда** (ни в заголовках, ни в теле):

| ❌ Не пишем | ✅ Что вместо |
|---|---|
| Команда профессионалов | «Юристы бюро» / конкретные имена |
| Индивидуальный подход | Описание того, что именно делаем под клиента |
| Заботимся о клиентах / клиенто­ориентированы | Удалить, не заменять |
| Качественные услуги / высокое качество | Конкретный результат или метрика |
| Широкий спектр / комплексные решения | Перечисление 3–5 услуг словами |
| Опытные специалисты | «Опыт — 12 лет», конкретно |
| Быстро / оперативно / в кратчайшие сроки | Срок в днях/неделях/месяцах |
| Доступные цены | Цифра или диапазон |
| На рынке с 2014 года | «Бюро основано в 2014 году» |
| Помогаем во всех вопросах | Перечислить разделы |
| Эффективные решения | Метрика результата |
| Профессиональный подход | Удалить |
| Доверьте нам / Доверяют нам | Только в подписи «Нам доверяют» к лого |
| Лучшие в своём деле | Удалить |

Если слово появилось в черновике — оно либо удаляется, либо заменяется на конкретику.

## 3. Цифры

### 3.1 Правила оформления

- Только конкретные. Никаких «много», «значительно», «существенно», «более десяти».
- Денежные суммы: `12,4 млн ₽` / `35 000 ₽` / `1,2 млрд ₽`. Запятая (не точка) для десятичных. Пробел (неразрывный, `&nbsp;` — поставит разработчик) каждые 3 разряда: `35 000`, не `35000`.
- Проценты: `96%` без пробела перед `%`.
- Сроки: `6–10 мес.` / `8 месяцев` / `2–3 недели`. Не «полгода», не «около года».
- Годы: `2024` / `2014–2026` / `с 2014 года`.
- Количества: `1 800+ знаков` / `~80 млн ₽` (тильда — только для оценок «примерно»).
- Тире между диапазоном — **длинное** `–` (en-dash), не дефис `-`. `6–10`, не `6-10`.

### 3.2 Когда цифры не известны

- Если точной цифры нет, но порядок — `~80 млн ₽`, `несколько десятков`. Только если порядок реально известен.
- Если порядок не известен — переформулировать без цифры. Лучше пропустить, чем выдумать.

## 4. Заголовки и italic-акценты

На сайте действует визуальный паттерн: в каждом заголовке (H1, H2 страниц, заголовки секций) **одно-два слова выделяются курсивом** — это смысловой акцент, отдельный шрифт + коричневый цвет.

### 4.1 Что выделять курсивом

- **Главная мысль заголовка** — то, ради чего читатель сюда пришёл.
- Существительное или короткая именная группа, не глагол и не союз.

Примеры:

| Заголовок | Курсив на чём |
|---|---|
| Регистрация товарного знака *в Роспатенте* | где |
| Партнёр в *сложных делах* | в каких |
| Защищаем клиентов по всей *России* | где |
| Взыскание компенсации *за нарушение прав* на товарный знак | за что |
| Что изменилось в экспертизе ТЗ в 2026 году: *разбор практики Роспатента* | что внутри |

### 4.2 Как обозначать в брифе

В тексте, который вы передаёте — italic пишется **звёздочками**:

```
H1: Регистрация товарного знака *в Роспатенте*
```

WP-разработчик заменит звёздочки на тег `<em>`. Подчёркивания (`_текст_`) не используем.

### 4.3 Длина заголовков

- **H1** — 40–80 символов с пробелами. Должен влезать в две строки на десктопе.
- **H2 секции на странице** — 20–60 символов. Должен влезать в одну строку.
- **H3 внутри секции** — 20–80 символов.

Слишком длинный H1 переползает в 3 строки и ломает hero — текст обрезайте, а длинное описание идёт в lead-абзац под заголовком.

---

## 5. Lead-абзац (под H1)

На каждой content-странице под H1 идёт **lead** — 1–3 предложения, 150–400 символов. Этот абзац — главное, что прочитает пользователь после заголовка.

### 5.1 Что должно быть в lead

- **Кому подходит** (для услуги/направления): «малому и среднему бизнесу», «производителям FMCG», «изобретателям и стартапам в фарме».
- **Что именно мы делаем** (без воды): не «помогаем», а «регистрируем», «защищаем в суде», «оспариваем доначисления».
- **Что отличает** или **что в результате** (опционально): «без отказов в 96% заявок», «решение устояло в кассации».

### 5.2 Антипримеры

❌ «Наша команда профессионалов поможет вам зарегистрировать товарный знак быстро и качественно.»
✅ «Регистрируем товарные знаки в Роспатенте: предварительный поиск, заявка, ведение делопроизводства. Подходит бизнесу, выводящему новый бренд за 6–10 месяцев.»

## 6. Описание клиентов в кейсах

Имя клиента можно указывать **только с письменным согласием**. По умолчанию — анонимизируем:

| Сектор клиента | Как описывать |
|---|---|
| FMCG | «производитель FMCG-товаров из ТОП-50» |
| IT-стартап | «IT-стартап с выручкой ~80 млн ₽» |
| Производство | «российский производитель промышленного оборудования» |
| Импортёр | «крупный импортёр электроники, оборот ~3 млрд ₽» |
| Сеть | «федеральная торговая сеть, 200+ магазинов» |
| Физическое лицо | «частный изобретатель в области медицины» |

**Что НЕ пишем:** «один из наших клиентов», «известный бренд», «лидер рынка». Это размытые формулировки, они уменьшают доверие.

## 7. Цены

Три формата:

| Когда | Формат | Пример |
|---|---|---|
| Есть минимум | `от X ₽` | `от 35 000 ₽` |
| Есть диапазон | `X – Y ₽` | `35 000 – 90 000 ₽` |
| Только индивидуально | `по запросу` | `по запросу` |

Никогда: «доступные цены», «выгодная стоимость», «оптимальная цена», «договорная».

**Государственные пошлины** указываем отдельной строкой от услуг бюро — клиент должен видеть, что пошлины платит государству, а не нам:
> Услуги бюро — от 35 000 ₽
> Пошлина за регистрацию заявки — 3 500 ₽
> Пошлина за экспертизу (1 класс МКТУ) — 11 500 ₽
> **Итого: от 58 100 ₽**

## 8. SEO-поля (title, meta description)

Каждая страница имеет два SEO-поля. Заполнять обязательно, разработчик ставит как есть.

### 8.1 `<title>`

- **До 65 символов** включая пробелы. После 65 поисковики обрезают «…».
- Шаблон: `{Тема страницы} · Галифанов, Мальков и Партнёры`
  Примеры:
  - `Регистрация товарного знака · Галифанов, Мальков и Партнёры`
  - `Патенты в медицине — оформление и защита · Галифанов, Мальков и Партнёры`
- На главной и hero-страницах можно отступать от шаблона.

### 8.2 Meta description

- **130–160 символов** включая пробелы. Меньше 120 — выглядит обрезанно, больше 160 — обрежет поисковик.
- Должно содержать: **что это + ключевая выгода + конкретика (цифра или срок)**.
- Не должно начинаться с «На этой странице…», «Здесь вы найдёте…» — это пустые слова.

Пример:
> Полный цикл регистрации товарного знака в Роспатенте: предварительный поиск, заявка, ведение делопроизводства. Сроки 6–10 мес., успешность 96%.

(157 символов)

### 8.3 Keywords

Не используем. Поисковики игнорируют этот meta-тег с 2009 года. Не тратим время.

## 9. Типографика

- **Кавычки русские: «ёлочки»**, внутри — „лапки“. Английские `""` и одинарные `''` не использовать.
- **Тире длинное** — `—` (m-dash) между предложениями и вместо «это»: «Бюро — это…». Не дефис `-`.
- **Тире средне** — `–` (en-dash) в диапазонах: `6–10 месяцев`, `2014–2026`. Не дефис.
- **Дефис** — `-` только в составных словах: «по-русски», «как-то», «когда-нибудь».
- **Многоточие** — символ `…`, не три точки `...`.
- **Знак №** перед числом: `№ 42`, `№ А40-128456/24`.
- **Знак ₽** — после числа с неразрывным пробелом: `35 000 ₽`.
- **Точка в конце заголовков** — не ставим.
- **Восклицательные знаки** — не используем ни в заголовках, ни в теле текста.
- **Сокращения**: «г.» (год), «гг.» (годы), «руб.» — лучше избегать, писать словом или символом ₽.

WP-разработчик прогонит финальный текст через скрипт типографики (неразрывные пробелы, замена кавычек и тире). Это не освобождает от того, чтобы писать сразу правильно — скрипт может пропустить.

## 10. Внутренние ссылки

В теле статей, кейсов, услуг можно и нужно давать ссылки на:

- Другие услуги бюро по теме (`/uslugi/...`)
- Существующие кейсы (`/dela/...`)
- Статьи журнала (`/zhurnal/...`)
- Классы МКТУ (`/mktu/klass-N/`)
- Юр. документы (`/legal/...`) — если упоминаем «обработку персональных данных», «оферту»

**Формат в брифе:** обычная markdown-ссылка `[текст ссылки](/uslugi/registratsiya-tovarnogo-znaka/)`.

**Не ставим:**
- Ссылки на внешние сайты, кроме первоисточников (kad.arbitr.ru, rospatent.gov.ru, законодательство на pravo.gov.ru).
- Ссылки на конкурентов.
- Ссылки в заголовках H1/H2/H3.

## 11. Фото и изображения

| Тип | Размер | Формат | Где используется |
|---|---|---|---|
| Cover статьи | 1600×700 (16:7) | JPG | `/zhurnal/{slug}/` |
| Cover кейса | 1600×900 (16:9) | JPG | `/dela/{slug}/` |
| Иллюстрация в статье | 1200×800 | JPG | внутри статьи |
| Портрет сотрудника | 1000×1000 (1:1) | JPG, фон `#F0EEEA` (paper-2) | `/o-byuro/#team` |
| Фото автора статьи | 200×200 (1:1) | JPG | под статьёй |
| Лого клиента | SVG, чёрный на прозрачном | SVG | главная, «Нам доверяют» |
| OG-image страницы | 1200×630 | PNG/JPG | автоматически по умолчанию |

**Что важно:**

- Все JPG — sRGB, качество 80–85%, без EXIF-метаданных.
- Лого клиентов — **строго SVG чёрного цвета**. На сайте фильтр инвертирует их под brand-фон.
- Портреты сотрудников — серый фон `#F0EEEA` (paper-2), плечи в кадре, освещение фронтальное. Фото на белом не подойдут — выбьются из карточек.
- Кейс/статья без cover-фото — можно сдавать, разработчик подберёт нейтральный placeholder. Но лучше своё.

## 12. Что прислать с каждым материалом

Любой контент-документ должен содержать в шапке:

```
Тип: услуга / кейс / статья / класс МКТУ / сотрудник / направление / специализация
Slug (URL): registratsiya-tovarnogo-znaka
Автор брифа: Иванова А.
Дата: 2026-05-15
Статус: черновик / на проверке / готов к публикации
```

Дальше — поля по шаблону соответствующего типа (`01–07-*.md`).

## Куда сдавать готовое

Папка на общем диске:
```
content/
  uslugi/
  dela/
  zhurnal/
  mktu/
  napravleniya/
  spetsializatsiya/
  komanda/
  photos/
  consents/        ← согласия клиентов на публикацию кейсов
```

Один документ на одну единицу контента (одна услуга = один файл). Имя файла = slug.

Финальный реципиент — WP-разработчик. Адрес и контакт — у заказчика.
