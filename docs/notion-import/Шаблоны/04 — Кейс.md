# Шаблон: кейс / дело

> Шаблон для **одного кейса** — конкретное дело, доведённое до результата.
> Кейсы наполняют главную, страницы направлений и каталог `/dela/`.
>
> **Эталон на сайте:** [/dela/vzyskanie-kompensatsii-fmcg/](https://1-tm.ru/dela/vzyskanie-kompensatsii-fmcg/)
> **URL результата:** `/dela/{slug}/`
>
> Прочитайте сначала [00-pravila-i-stil.md](00-pravila-i-stil.md), особенно §6 про
> анонимизацию клиентов.

---

## ⚠️ Перед написанием кейса

**Согласие клиента на публикацию — обязательно.** Либо есть письменное согласие
(с разрешением указывать имя), либо кейс анонимизируется до неузнаваемости.

Согласие сохраняем в `content/consents/` под slug-ом кейса.

Если согласия нет:
- Имя/название компании клиента — не пишем.
- Описываем сектор и масштаб: «производитель FMCG-товаров из ТОП-50».
- Не указываем уникальные детали, по которым клиент опознаётся (точный товар, регион, дата + сумма).
- Если результат публичный (карточка в `kad.arbitr.ru` с открытым реестром) — можно ссылаться на номер дела, но без имени клиента.

```
Тип: кейс

Направление: [интеллектуальная собственность / договоры и сделки / споры и защита / сопровождение бизнеса]
Автор брифа: [Фамилия И.]
Дата: [ГГГГ-ММ-ДД]
Статус: черновик
```

## 1. SEO-поля

### 1.1 `<title>` (до 70 символов)

Формула: `[Краткое название кейса] — [результат]`

**Примеры:**
- `Взыскание 12,4 млн ₽ компенсации за нарушение прав на ТЗ`
- `Защитили патент в СИП — иск конкурента отклонён в полном объёме`
- `Снято 47 млн ₽ доначислений по выездной налоговой проверке`

### 1.2 Meta description (130–160 символов)

```
[Что за кейс] + [ключевой результат] + [контекст или цифра].
```

**Пример:**
`Взыскали 12,4 млн ₽ компенсации за нарушение прав на товарный знак в FMCG. Решение устояло в апелляции и кассации. Дело А40-128456/24.`

## 2. Hero кейса

### 2.1 Категория (1–3 категории через «·»)

```
[Направление] · [Раздел] · [Сегмент]
```

**Примеры:**
- `Товарные знаки · Интеллектуальная собственность`
- `Налоговые споры · Сопровождение бизнеса`
- `Договоры · Франшиза`

### 2.2 Номер дела (опционально, но желательно)

```
А40-128456/24
```

Форматы:
- Арбитраж: `А40-128456/24` (А — округ, 40 — Москва), `А55-…` (Самара) и т.д.
- СИП: `СИП-1042/2024`
- Роспатент / Палата: `№ РД0042781`, `№ 2024Б00347`
- Верховный Суд: `№ 305-ЭС24-…`

Если согласия нет и номер раскрывает клиента — не указывайте.

### 2.3 Юрисдикция (2–4 слова)

```
[Суд / Орган] · [Город]
```

**Примеры:** `Арбитраж · Москва`, `СИП · Москва`, `Роспатент · 2024`, `ФАС России · Москва`

### 2.4 Период (год или диапазон)

```
2024 / 2024–2025
```

### 2.5 H1 заголовка кейса (40–80 символов, italic-акцент)

```
[Действие] *[акцент]* + [контекст]
```

**Примеры:**
- `Взыскание компенсации *за нарушение прав* на товарный знак в FMCG`
- `Защита патента *на медицинское устройство* в СИП`
- `Снятие доначислений *по выездной проверке* за 2021–2023`

### 2.6 Метка результата (1–3 слова)

Короткая, однозначная.

**Варианты:** `Взыскано`, `Снято доначислений`, `Зарегистрировано`, `Возвращены права`, `Признан недействительным`, `Иск отклонён`, `Договор расторгнут`.

### 2.7 Значение результата (короткое)

**Варианты по типу кейса:**

| Тип результата | Формат | Пример |
|---|---|---|
| Денежный | `X млн ₽` / `X тыс. ₽` | `12,4 млн ₽` |
| Процент | `X%` | `100%` (полное удовлетворение) |
| Количество | `N` | `7` (знаков зарегистрировано) |
| Срок | `N мес.` | `8 мес.` |

### 2.8 Описание результата (100–180 символов, 1 предложение)

```
[Конкретный исход + что устояло / что отменено].
```

**Пример:**
`В полном объёме удовлетворено требование о двукратной стоимости. Решение устояло в апелляции и кассации.`

## 3. H2: «Задача» — что нужно было решить

**Длина:** 200–500 слов. 1–3 абзаца + опционально маркированный список из 3–5 пунктов «Что хотел клиент».

**Что писать:**

1. **Контекст клиента** (анонимизированный): сектор, размер, тип бизнеса.
2. **Что произошло** до обращения: какое событие запустило дело.
3. **Boundary-условия:** сроки, бюджет, ограничения по PR, рыночные риски.

**Что НЕ писать:**
- Не давать оценок «клиент сделал глупость» / «контрагент действовал злонамеренно».
- Не пересказывать общие положения закона.
- Не обещать («мы сразу поняли, что выиграем»).

**Заполните:**
```
[Абзац 1 — кто клиент и что произошло, 80–150 слов]

[Абзац 2 — что от нас хотели, 80–150 слов]

Что хотел клиент:
- [результат 1]
- [результат 2]
- [ограничение 1, например, по срокам]
```

## 4. H2: «Решение» — как мы вели дело

**Длина:** 300–700 слов. 2–4 абзаца + список из 4–8 ключевых действий.

**Что писать:**

1. **Стратегия:** какую линию выбрали и почему. Какие альтернативы рассматривали.
2. **Ключевые действия:** что именно делали — ходатайства, экспертизы, документы.
3. **Препятствия:** что усложняло (позиция оппонента, требования суда, риск встречного иска) и как преодолевали.
4. **Профессиональные термины** допустимы — аудитория кейсов профильная (юристы, директора).

**Заполните:**
```
[Абзац 1 — стратегия, 100–200 слов]

[Абзац 2 — ключевые действия, 100–200 слов]

[Абзац 3 — препятствия и решения, 100–200 слов]

Ключевые шаги:
- [шаг 1 — конкретное действие]
- [шаг 2 — конкретное действие]
- [шаг 3 — ...]
- [...]
```

---

## 5. H2: «Результат» — к чему пришли

**Длина:** 100–300 слов + блок с 3 метриками.

**Что писать:**

1. **Основной исход** в первом абзаце.
2. **Сопутствующие эффекты** во втором (если есть): доля рынка, репутация, прецедент.
3. **3 метрики** в блоке (название + значение).

**Заполните:**
```
[Абзац 1 — основной результат и что он значит для клиента, 80–150 слов]

[Абзац 2 — сопутствующие эффекты, опционально, 50–150 слов]

Метрики:
1. [Название метрики] — [значение]
2. [Название метрики] — [значение]
3. [Название метрики] — [значение]
```

**Пример блока метрик:**
1. Взыскано — 12,4 млн ₽
2. Срок дела — 8 мес.
3. Снижений в инстанциях — 0

## 6. Опционально: иллюстрации внутри кейса

Можно вставлять одно-два фото/скриншота внутри разделов «Задача» / «Решение» / «Результат». Это улучшает восприятие лонгрида.

Подходит:
- Фото зала суда (анонимное)
- Скан фрагмента решения (с замазанными именами)
- Схема процесса / тайм-лайн
- График/диаграмма с метриками

Не подходит:
- Стоковые фото «юристы за столом»
- Просто красивые картинки без смысла

Формат — JPG 1200×800, без EXIF.

## 7. Похожие дела (внизу страницы)

Внизу автоматически идёт блок «Похожие дела» — 3 карточки. Можете предложить slugs:

```
Похожие дела:
- /dela/[slug-1]/
- /dela/[slug-2]/
- /dela/[slug-3]/
```

Если не уверены — пометьте «подбирает редактор».

## 8. CTA-final (текст призыва внизу страницы)

Призыв должен быть **по теме кейса**, не общий.

**Пример (для кейса о взыскании компенсации):**
```
Eyebrow: Похожая ситуация?
H2: Считаем компенсацию *за нарушение прав*
Lead: Расскажите о ситуации — оценим перспективы взыскания, рассчитаем диапазон
суммы. Первая консультация бесплатна.
```

**Пример (для кейса о налоговом споре):**
```
Eyebrow: Получили акт проверки?
H2: Снимаем доначисления *в досудебном и судебном порядке*
Lead: Пришлите акт — за 2 рабочих дня разберём по эпизодам, оценим перспективы
снижения и риски. Первая консультация бесплатна.
```

1. **Согласие клиента** (если есть имя): PDF в `consents/`.

3. **Опционально — иллюстрации** для тела кейса (см. §6).
4. **Ссылка на дело в публичных реестрах** (если открытое): kad.arbitr.ru / fips.ru.
