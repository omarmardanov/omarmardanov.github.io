## 1. SEO-поля

### 1.1 `<title>` (до 65 символов)

```
[Название направления] — услуги бюро · Галифанов, Мальков и Партнёры
```

**Пример:** `Договоры и сделки — услуги бюро · Галифанов, Мальков и Партнёры`

### 1.2 Meta description (130–160 символов)

```
[Что за направление в 1 фразе] + [для кого] + [перечисление 2–3 ключевых услуг].
```

**Пример:**
`Юридическое сопровождение договоров для бизнеса: подготовка и проверка договоров, due diligence сделок, договоры франшизы и лицензии. Москва и Тюмень.`

---

## 2. Hero страницы

### 2.1 Eyebrow (3–5 слов)

```
Направление работы
```

(на всех страницах направлений одинаково)

### 2.2 H1 (40–80 символов, с italic-акцентом)

```
[Название направления] *[акцент]*
```

**Примеры:**
- `Договоры и сделки *для бизнеса*`
- `Споры и защита *в судах*`
- `Сопровождение бизнеса *под ключ*`

### 2.3 Lead-абзац под H1 (200–400 символов)

```
[Что входит в направление в целом + для кого + что отличает].
```

**Пример (Договоры и сделки):**
`Готовим и проверяем договоры, сопровождаем M&A и крупные сделки, оформляем франшизы и лицензии на интеллектуальную собственность. Работаем с компаниями от стартапов до групп с оборотом 10+ млрд ₽.`

### 2.4 Aside-nav (содержание, 3–6 пунктов)

Совпадает с H2-заголовками разделов внутри направления.

**Пример:**
1. Корпоративные договоры
2. M&A и сделки
3. Франшиза
4. Лицензии и роялти

## 3. Разделы внутри направления (3–6 разделов)

Каждое направление содержит несколько **разделов**, в каждом разделе — несколько **услуг**.

### Структура одного раздела

```
H2: [Название раздела] *[опц. italic-акцент]*

Краткое описание раздела (50–150 символов, 1 предложение):
[Что входит в этот раздел.]

Услуги (4–10 штук):
- [Услуга 1] — [короткое описание, 50–100 символов]
  Slug: [/uslugi/{направление}/{раздел}/{slug}/]
- [Услуга 2] — [...]
  Slug: [...]
```

### Пример: раздел «Корпоративные договоры»

```
H2: Корпоративные договоры

Описание: Подготовка и проверка договоров для повседневной работы бизнеса —
от поставки до коммерческой концессии.

Услуги:
- Договор поставки — подготовка с защитой ваших интересов, риск-анализ
  типовых пунктов контрагента.
  Slug: /uslugi/dogovory-i-sdelki/korporativnye/dogovor-postavki/

- Договор оказания услуг — проработка предмета, ответственности, порядка
  приёмки, защита от навязанных правок.
  Slug: /uslugi/dogovory-i-sdelki/korporativnye/dogovor-uslug/

- Договор подряда (строительный/IT) — учёт специфики отрасли, защита
  от рисков просрочки и качества.
  Slug: /uslugi/dogovory-i-sdelki/korporativnye/dogovor-podryada/

- ...
```

## 4. Связанные кейсы (внизу страницы)

3 кейса по теме направления. Указать **slugs** существующих кейсов:

```
Связанные дела:
- /dela/[slug-1]/
- /dela/[slug-2]/
- /dela/[slug-3]/
```

Если ещё не понятно какие — пометьте «подбирает редактор».

---

## 5. CTA-final (текст призыва внизу страницы)

```
Eyebrow: [3–6 слов, моно-метка]
H2 с italic-акцентом: [сильное обещание или прямой вопрос]
Lead: [2–3 предложения, что произойдёт после клика — срок ответа, бесплатность консультации]
```

**Пример (для «Споры и защита»):**
```
Eyebrow: Защита бизнеса в судах
H2: Готовим *стратегию защиты* до подачи иска
Lead: Расскажите о ситуации — за 1 рабочий день оценим перспективы, риски и предложим
стратегию. Первая консультация бесплатна.
```

## 6. Чек-лист

- [ ] Шапка документа заполнена
- [ ] `<title>` ≤ 65 символов
- [ ] Meta description 130–160 символов
- [ ] H1 с italic-акцентом через `*звёздочки*`
- [ ] Lead 200–400 символов
- [ ] Aside-nav 3–6 пунктов, совпадает с H2-разделами
- [ ] 3–6 разделов, в каждом 4–10 услуг
- [ ] Все услуги имеют slug (URL)
- [ ] Slugs услуг латиницей, в kebab-case (`-`)
- [ ] 3 связанных кейса предложены или помечены «подбирает редактор»
- [ ] CTA-final текст прописан под контекст направления
- [ ] Прошёл по чёрному списку штампов (правила §2)
